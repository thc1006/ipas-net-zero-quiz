> 更新於 **2026-07-14**。
> 目前的完整缺口清單在 [`VERIFICATION-GAPS.md`](VERIFICATION-GAPS.md)（由 `tools/gen_gap_reports.py`

## 第二輪的結論是錯的

第二輪結束時，我列出 **34 題「拿不到可驗證的一手證據」**，分成六類，其中最大的兩類是：

- **A 類：標準本文付費**（8 題）—— ISO / PAS 的正文在付費牆後，`iso.org` 只給摘要。
  當時我寫的結論是：**「買標準才驗得了。這是真正的能力邊界。」**
- **C 類：一手來源不會這樣寫**（14 題）—— 概念題、主觀排序；法規不會寫「最常見的挑戰是…」。
  當時我寫的結論是：**「這類題目本身可能就有問題。」**

**這兩個結論都是錯的。**

## 錯在哪：我一直沒去搜這場考試自己的課本

iPAS 的命題依據是**公版教材** —— 免費、公開，掛在中華大學（`usr.chu.edu.tw`）
與 `ipas.org.tw` 上。ISO 標準的正文確實要錢，
**但公版教材把要考的條文逐條翻成中文寫在裡面**，
而且好幾份 PDF 的最後直接印著**範例題與答案卡**。

第三輪把「**先窮盡搜公版教材**」寫進代理契約，再把那 34 題重新派出去：

| | 第二輪的判定 | 第三輪的結果 |
| --- | --- | --- |
| A 類（標準付費） | 「買標準才驗得了」 | **在公版教材裡找到逐字依據** |
| C 類（來源不會這樣寫） | 「這類題目本身可能有問題」 | **多數在公版教材的範例題裡找到原題與答案卡** |
| 合計 34 題 | 「無法查證」 | **32 題拿到通過機械驗證的逐字引文** |

**「無法查證」很多時候只是「我沒去對的地方找」。**
把這句話寫進契約，比派更多代理有用得多。

## 那個「更大的缺口」也查完了 —— 而且結果比預期糟

曾經有 **383 題「有一手來源 URL，但從來沒有人打開來看過」**。
第四輪把這 383 題**逐題把那一頁抓回來**，檢查它到底有沒有說這件事：

| 複驗結果 | 題數 |
| --- | ---: |
| 引用正確 | 294 |
| **引錯地方 —— 那一頁根本不含這個主張** | **31** |
| 主題相關，但沒有一句話釘得住答案 | 35 |
| 連結已死（含**軟 404**：回 200，內文卻寫「您所查詢的網址不存在」） | 4 |

**引錯的那 31 題，URL 全部都回 HTTP 200。連結健康檢查一個都抓不到。**

典型的錯法：問「**核能是不是再生能源**」的題目，引用指向 **CBAM 法規**（全文 0 次 "nuclear"）；
問 **PAS 2060** 的題目，引用指向 `iso.org/standard/43279.html` —— 那是 **ISO 14068-1**。

**「有來源」與「有人看過內容」是兩件事，而且差距比想像中大得多。**
現在每一題的複驗結果都記在 `citation_audit` 裡；
還沒有逐字引文的只剩 **82 題**（詳見 [`VERIFICATION-GAPS.md`](VERIFICATION-GAPS.md)）。
