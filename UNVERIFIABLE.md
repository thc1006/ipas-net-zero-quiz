# 「無法查證」的題目 —— 第三輪之後

> 更新於 **2026-07-14**。
> 目前的完整缺口清單在 [`VERIFICATION-GAPS.md`](VERIFICATION-GAPS.md)（由 `tools/gen_gap_reports.py`
> 從資料生成）。這一份記的是**「無法查證」這件事本身的變化** —— 它比我想像中容易解決得多。

## 第二輪的結論是錯的

第二輪結束時，我列出 **34 題「拿不到可驗證的一手證據」**，分成六類，其中最大的兩類是：

- **A 類：標準本文付費**（8 題）—— ISO / PAS 的正文在付費牆後，`iso.org` 只給摘要。
  當時我寫的結論是：**「買標準才驗得了。這是真正的能力邊界。」**
- **C 類：一手來源不會這樣寫**（14 題）—— 概念題、主觀排序；法規不會寫「最常見的挑戰是…」。
  當時我寫的結論是：**「這類題目本身可能就有問題。」**

**這兩個結論都是錯的。**

## 錯在哪：我一直沒去搜這場考試自己的課本

iPAS 的命題依據是**公版教材** —— 免費、公開，掛在中華大學（`usr.chu.edu.tw`）
與 `ipas.org.tw` 上。ISO 標準的正文確實要錢，
**但公版教材把要考的條文逐條翻成中文寫在裡面**，
而且好幾份 PDF 的最後直接印著**範例題與答案卡**。

第三輪把「**先窮盡搜公版教材**」寫進代理契約，再把那 34 題重新派出去：

| | 第二輪的判定 | 第三輪的結果 |
| --- | --- | --- |
| A 類（標準付費） | 「買標準才驗得了」 | **在公版教材裡找到逐字依據** |
| C 類（來源不會這樣寫） | 「這類題目本身可能有問題」 | **多數在公版教材的範例題裡找到原題與答案卡** |
| 合計 34 題 | 「無法查證」 | **32 題拿到通過機械驗證的逐字引文** |

**「無法查證」很多時候只是「我沒去對的地方找」。**
把這句話寫進契約，比派更多代理有用得多。

| 其他 | 23 | 窮盡搜過公版教材、法規、環境部與國際標準仍找不到 —— 多半是社群共筆自行改寫、不對應任何官方段落的題目 |

逐題清單見 [`VERIFICATION-GAPS.md`](VERIFICATION-GAPS.md)。

## 還有一個更大的缺口，不要忽略它

**383 題「有一手來源 URL，但沒有人逐字確認引文真的在那一頁上」。**

⚠️ **連結是活的，不代表指對地方。** 這個專案抓到過 **19 題引錯法規** ——
碳費的題目引到溫管辦法，兩個 URL 都回 HTTP 200，連結健康檢查完全看不出來。

「有來源」與「有人看過內容」是兩件事。README 的三級來源表把它們分開列，就是為了這個。
